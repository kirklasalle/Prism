🛡️ PRISM Operator Dashboard Walkthrough

Dashboard UI Manual Step-by-step walkthrough for configuring Sovereign Sentinel (SSHP) and managing Cognitive Session Handoff (CSH) "Baton Pass" loops.

1. Dashboard Tab Layout Reference

Navigating the PRISM dashboard is partitioned into four primary operator screens:

💬 Chat Tab
Central natural language prompt interface to task agents.
🖥️ Computer Tab
Local hardware telemetry, live Vision Framebuffer, terminal console, and demos.
🌐 Browser Tab
Zero-trust embedded browser with active security proxy settings.
🤖 Agentic Tab
Tracks goal plans, live execution outputs, and human handoff cues.

2. Step-by-Step Operator Playbook

Step 1

Verify SSHP Security Settings

Before launching an autonomous browser-use agent, make sure privacy redaction is active:

  1. Go to the Browser or Computer tab.
  2. Locate the Sovereign Sentinel (SSHP) Shielding panel under settings.
  3. Ensure PII Redaction Mode is toggled to ENABLED.
  4. SSHP will scrub emails, cards, and SSNs from both screenshots (using visual blackout boxes) and DOM trees.
Step 2

Submit an Autonomous Goal

Trigger the closed-loop agent loop to accomplish a web or desktop task:

  1. Go to the Computer Tab and locate 🤖 Autonomous Control.
  2. Type in your high-level natural language prompt. Example:
    "Search GitLab for prism pull requests and list their active reviews."
  3. Select allowed tools: Browser Use, Computer Use, Shell Exec.
  4. Click ▶ Execute.
Step 3

Handle CAPTCHA or Security Suspension Cues

If an external site triggers a security wall, the agent halts its FSM loop automatically:

1. running
──►
2. handing_off
──►
3. suspended
  1. The agent writes its cookie jars, localStorage, sessionStorage, and plan context to a state pack.
  2. The FSM enters a suspended state, halting execution safely.
  3. An operator warning will flash in the Agentic Tab.
Step 4

Take the Baton & Solve roadblock

As the human operator, you can now bypass the roadblock in under a minute:

  1. Navigate to the Agentic Tab and find Pending Handoffs.
  2. Click 🤝 Take the Baton.
  3. An interactive browser frame will open, populated with the agent's exact cookie session and page coordinates.
  4. Resolve the roadblock (e.g. solve the CAPTCHA, log in via OAuth/MFA, or bypass the landing block).
Step 5

Hand Back Baton and Resume

Return the session back to the AI loop to complete the remainder of the task:

  1. Click ◀ Hand Back Baton in your dashboard frame.
  2. The CSH Manager serializes your updated session variables back into the execution container.
  3. The agent FSM switches back to running and finishes the task seamlessly.

Q: Can I run multiple handoffs simultaneously?
