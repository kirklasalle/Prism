1. Executive Launch Context

Going public on GitHub is a pivotal event for PRISM. Because PRISM's primary market differentiator is cryptographically verified, decoupled Governance-as-a-Service (GaaS), any public launch must demonstrate flawless codebase safety, robust security postures, and pristine developer ergonomics.

This launch preparation roadmap merges the recommendations surfaced by our May 2026 World-Class Project Audit with the active Q2 Roadmap. The checklist ensures all developer pipelines are green, zero credentials leak, and external developers can bootstrap in under 15 minutes.

Stage A

Security & Cryptographic Sweep

⚠️
Critical Security Caution: A single leaked API key in your public Git history can compromise your corporate accounts, ruin trust, and trigger automated security takedowns.
Scan Historical Git Objects Run trufflehog or gitleaks locally over the entire commit history to verify no raw tokens remain.
Scrub History on Leaks Detection Use git-filter-repo to programmatically purge sensitive files or variables from all historical trees.
Validate .gitignore Coverage Confirm that *.db, workspace/logs/, .env, *.pem, and config/plugin-signing-keys.json are fully blocked.
Sanitize .env.example Provide a comprehensive env example template with comments explaining all 25 variables.

Legal & Open-Source Foundations

License Selection Place an Apache 2.0 or AGPL-3.0 License file in the repository root to establish commercial boundaries.
Security Policy Setup Create SECURITY.md detailing private contact emails for responsible vulnerability disclosures.
Contribution Rules Draft CONTRIBUTING.md outlining style requirements and pull-request build verification mandates.
Signed Directives Invariant Documentation Document the cryptographic 10 Laws boot-checking mechanism to inform prospective PR submitters.
Stage C

Rigorous Testing & CI/CD Validation

Verify Mock-Fallback Environments Assert that integration tests skip or trigger mocks gracefully if node-pty or Docker is missing locally.
Implement E2E Smoke Tests Commit a 60s tests/e2e-smoke.test.ts verifying auth, chat, tool mock, and policy interception.
Create GitHub Actions Workflow Commit .github/workflows/ci.yml covering lints, builds, unit-tests, and release qualification checks.
Prebuild Qualification hook Wire npm run prebuild as a pre-commit check to prevent compiling outdated directive hashes.

# Check for uncommitted database, log, or key files git status --ignored
