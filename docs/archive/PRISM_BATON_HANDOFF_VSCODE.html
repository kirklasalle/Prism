🛠️ Phase 1: Completed — Paradigm Switcher Persistence

The Problem: The operator engaged 🔍 Auto-Detect Mode using a local llama.cpp model. However, the paradigm switcher controls in the upper-left brand sidebar kept vanishing on every state update tick.

The Discovery & Root Cause: The regular dashboard rendering loop executed renderBrandPanel() inside src/core/operator/public/tab-chat.js. This function replaced the innerHTML of #brand-panel entirely, wiping out the paradigm controls added by the static HTML template.

The Resolution: Reconstructed the brand generator loop to read the existing DOM states (logs, badge selections, active mode toggles) and dynamically reconstruct the fully interactive ⚡ Base, 🚀 Frontier, and 🔍 Auto paradigm panels natively inside renderBrandPanel().

// File: src/core/operator/public/tab-chat.js (around line 1046) var existingBadge = document.getElementById('prism-paradigm-badge'); var badgeText = existingBadge ? existingBadge.innerText : 'LOADING'; // State is extracted from DOM, and paradigm panel HTML is appended directly inside brand.innerHTML!

🎯 Phase 2: Next Critical Step — Wiki Sidebar Navigation Spacing

Operator Bug Report: When the browser window is maximized, the Wiki navigation panel displays excessive vertical space between list items. In non-maximized viewports, item spacing remains tightly and correctly configured.

Target Assets: src/core/operator/public/tab-wiki.html and tab-wiki.js.

Engineering Analysis: The parent container .wiki-container employs align-items: stretch. The sidebar drawer aside.wiki-sidebar-drawer has a flex column layout. When maximized, the height expands, causing inner tree elements or the list wrapper #wiki-sidebar-list to grow or apply vertical justification across the larger viewport.

🔧 Copilot Implementation Guide:

  • Open src/core/operator/public/tab-wiki.js. Ensure categories are rendered with style.flex = 'none' and style.height = 'auto'.
  • Verify #wiki-sidebar-list has justify-content: flex-start !important; to top-align items.
  • Make sure category collapsible items (A-Z tree) do not have flex: 1 applied.
