🎬 PTAC Operator Demo

v0.20.1 One-click, end-to-end self-driven recording of the PRISM dashboard.

What it is

PRISM drives its own dashboard — chat, approval queue, computer-use, browser, real PTY, real Docker — capturing every step as evidence and rendering the result as a portable, browser-playable HTML slideshow.

Three gates

The demo is triple-gated. All three must be set to 1 before the button activates:

① PRISM_PTAC_OPERATOR_DEMO=1 — admin opt-in (the panel itself).
② PRISM_PTAC_SAFE=1 — host-prepared confirmation (per session).
③ PRISM_PTAC_RECORD_VIDEO=1 — recording opt-in (per session).

Quick start

$env:PRISM_PTAC_OPERATOR_DEMO = "1"
$env:PRISM_PTAC_SAFE = "1"
$env:PRISM_PTAC_RECORD_VIDEO = "1"
.\start_web.bat
  1. Open the dashboard, go to Computer Control.
  2. Scroll to 🎬 PTAC Operator Demo. The status pill should read ready.
  3. Pick a suite: demo (curated), fast (~2 min), or full (~10 min).
  4. Click ▶ Start Recorded Run. The runs list polls for new output every 5 seconds.
  5. When your run appears, click ▶ Slideshow to replay it inline.

What you get

File What it is
video.html Self-contained slideshow with play / pause / prev / next / restart. Inlines all frame metadata; loads PNGs relatively so the run folder is portable.
video-manifest.json { runId, fps, frameCount, durationSec, frames[] }
report.html Full PTAC HTML report with per-step evidence.
summary.json Run-level result tree.
steps.jsonl / events.jsonl Per-step + per-event JSON-Lines logs.
screenshots/ One PNG per recorded frame, content-addressed by SHA-256 prefix.
transcript.json / transcript.txt Deterministic narration (when --demo-recording).

Why slideshow and not MP4?

Encoding video would require ffmpeg or a wasm encoder — both violate PRISM's zero-new-runtime-deps invariant. The slideshow is a single static HTML page; any screen recorder pointed at it will produce a true MP4 if needed downstream.

CLI alternative

PRISM_PTAC_SAFE=1 PRISM_PTAC_RECORD_VIDEO=1 \
  node dist/src/ptac/cli.js \
    --profile=sandbox \
    --suite=demo \
    --demo-recording \
    --record-video \
    --record-video-fps=2

🛡️ Sovereign Sentinel Hyper-Proxy (SSHP)

active Programmatic visual redactor, DOM scrubber, and Sacred Covenant integrity scanner.

Security Features

  • Visual PII Masking: Draws high-accuracy black rectangles directly onto screenshots when sensitive elements are focused (e.g. passwords, card CVVs, private keys).
  • DOM Sanitization: Scans and wipes email formats, credit cards, and SSNs from HTML DOM trees before models can ingest them.
  • Sacred Covenant Enforcement: Pre-scans actions (clicks, types, navigations) to block unsafe URL protocols (e.g., file:///etc/passwd) or destructive evaluations (e.g., localStorage.clear()).

Configuration API

Method Path Description
POST /api/preferences/sshp-redaction Toggle SSHP PII redaction on/off dynamically. Default: enabled: true.
